Week 5: Enumerable Methods (Map) January 31st, 2016
Hello again! Today we will be learning about a very useful Ruby method - the method ".map". This is one of the most valuable methods I have encountered for manipulating data structures.

This method would typically be used on an array in order to return a new array. This is most useful when you have items stored in an array, and want to manipulate each element of the array in a particular way. Let's explore this with a simple mathematical example.

Take the the array [1, 2, 3, 4, 5]. Let's say you wanted to return a new array storing the square of each number - [1, 4, 9, 16, 25]. We could easily use the map method here: 

    [1, 2, 3, 4, 5].map {|number| number ** 2}
This would return [1, 4, 9, 16, 25]. How did that work? Well, the "number" variable between the vertical lines represents each element in the array. Then, the block of code afterwards tells Ruby how we want it to manipulate the element. Finally, we get a new array with our results.

The .map method can also be used on strings. Let's say we had a task to define a method, pluralize, that added an "s" to the end of any given word. We could easily use .map to write this method in one line of code. 

    def pluralize(array)
      array.map {|word| word + "s"}
    end

Testing that method like so: 

    pluralize(["dog", "cat", "rabbit"])
The method returns a new array ["dogs", "cats", "rabbits"].

Hopefully this is making sense to you! If you are interested in learning more about .map and other Enumerable methods, I again suggest that you take a look at the Ruby Docs. Happy mapping!
